# 実行指示書: 南原竜樹 マーケット解説ショート動画（slot1）

ローカルの Claude Code（このリポジトリを開いた状態。`.mcp.json` で Higgsfield MCP 接続済み）に
「`prompts/run_nambara_slot1.md` を読んで、その通りに実行して」と一言伝えるだけで開始できます。
長い指示をターミナルに貼り付けると画面が固まるため、指示はこのファイルに集約しています。

---

## ゴール
「南原竜樹／南原たつき」本人の現在の顔で、マーケット解説のショート動画を作り、
`media/nambara_slot1.mp4` に保存。プレビュー確認後に X 投稿（GitHub Actions 経由）まで進める。

## 最重要・顔参照（厳守）
- 本人の「現在の」顔写真**5枚だけ**を唯一の参照に使う（正面・右斜め・横・正面もう1枚・後ろ姿）。
- 写真のローカルパス（実行者が記入）:
  - `★ここに5枚のファイルパス、またはフォルダパスを記入★`
- ネット上の整形前の古い写真は顔の形が違うので**絶対に使わない**。上記5枚以外を参照にしない。

## 手順
1. **接続**: Higgsfield MCP に接続。未認証ならブラウザで OAuth 認証（`t.nambara.autotrading@gmail.com` でサインイン）。
2. **コスト先出し**: `balance` を確認し、各生成を `get_cost:true` で試算して合計を提示。
   コストが高い／エラー時は止めて原因と選択肢を提示。問題なければ進める。
3. **アップロード**: 上記5枚をローカルから Higgsfield へアップロード
   （`media_upload` で presigned URL 取得 → バイトを PUT → `media_confirm`）。得た `media_id` を控える。
4. **顔参照**: 5枚から顔参照を作成（`show_reference_elements` action=create、または Soul 学習）。
   本人の顔を厳密に再現できる方法を選ぶ。
5. **人物画像**: 正面写真ベースで「本人が正面で話している表情」を生成。
   - model: `nano_banana_2` / aspect: `9:16`
   - 横顔も参照して一貫性を上げる
   - 服装はグレースーツ系、背景はオフィス or 薄いマーケットチャート、落ち着いた印象
6. **ナレーション音声**: 日本語 TTS を生成。
   - model: `inworld_text_to_speech` / voice: `"Satoshi (ja)"`
   - 原稿（slot1）:
     「おはようございます、南原竜樹です。今日もマーケット情報をお届けします。
      今朝は為替と米国株の流れに注目。寄り付きの動きを一緒にチェックしていきましょう。」
7. **動画生成（顔同期）**:
   - model: `wan2_7` / aspect: `9:16`
   - `start_image` = 手順5の人物画像 / `audio` = 手順6の音声（口パク同期）
   - 下部に大きめテロップ
   - 尺は約20秒希望だが、口パク同期モデルは**最長15秒**。まず15秒で作る。
     20秒にしたい場合は2クリップに分けて繋ぐ案を提示してから判断する。
   - 顔は添付5枚の現在の顔を厳密に維持する。
8. **保存**: 完成動画を `media/nambara_slot1.mp4` としてダウンロード保存。
9. **プレビュー**: 動画ファイルを実行者に見せる。**OK が出るまで X には投稿しない。**
10. **X 投稿（GitHub Actions 経由）**: OK 後に実施。
    - `media/nambara_slot1.mp4` をコミット＆プッシュ
    - `posts/messages.json` の slot1 に `"media": "media/nambara_slot1.mp4"` を追加
    - 本文は `messages.json` の slot1 テキスト
    - ドラフト PR を作成。実際の投稿は secrets を持つ GitHub Actions で実行する。

## 進め方の原則
- 各生成の前にコストを出す。エラーや想定外は止めて実行者に確認する。
- 進捗は日本語で報告する。
- 参照画像は上記5枚のみ。古い写真・ネット画像は使わない。
